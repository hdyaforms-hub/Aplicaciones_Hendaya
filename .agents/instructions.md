# Persistent Project Instructions

## UI Design Guidelines (Required by System Administrator)

- **Overall UI Theme**: The application MUST follow a clean, light-colored aesthetic (white/gray-50 backgrounds). Do NOT use dark/black backgrounds for large UI blocks like titles or descriptions unless explicitly requested again.
- **Input Content Visibility**: All content that the user *types* or *selects* (especially in fields like Código, Fecha, and Versión) MUST be high-contrast black (`text-black`) and bold (`font-black` or `font-bold`).
- **Metadata Fields**: Fields for Code, Date, and Version should have clear borders (`border-gray-300`) and the text inside must be strictly black for maximum visibility.
- **Form Renderer**: All user responses in the form renderer should use black text to ensure they are easily readable against the light background.
- **Contrast Rationale**: The user has explicitly stated that light gray tones are difficult to see in their environment. This is a non-negotiable requirement for all new windows or components created.
